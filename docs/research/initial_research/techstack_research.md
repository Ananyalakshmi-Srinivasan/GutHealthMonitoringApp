### Tech stack research

This contains the table we used to collect data about the different possible technologies we could use for this project:

| **Technology / Layer**                             | **Description**                                                                                                                                                                                                                                         | **Fit to Project**                                                                                                                                                                                | **Maintenance**                                                                                                                             | **Risks**                                                                                 |
| -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------- |
| **Frontend – Flutter Framework**                   | An open-source UI toolkit maintained by Google that allows developers to build cross-platform applications for iOS and Android using a single Dart codebase. It provides native performance, a modern reactive UI system, and strong community support. | Excellent fit for multiplatform development — supports both iOS and Android natively with consistent design and fast performance. Built-in support for notifications, navigation, and animations. | Maintained through periodic SDK updates, plugin version control, and CI/CD automation.                                                      | Minor learning curve for Dart; performance depends on device optimization.                |
| **Frontend – React Native**                        | A cross-platform app framework maintained by Meta (Facebook) that allows building native apps using JavaScript and React. Provides access to native APIs and a rich ecosystem of UI components.                                                         | Suitable alternative to Flutter if JS/React skills dominate. Excellent for future scalability (e.g., adding a web version).                                                                       | Requires regular NPM package and framework updates. Can be managed via Expo for simplicity.                                                 | Dependency conflicts and version mismatches may occur; slightly slower than native apps.  |
| **Backend – Java (Spring Boot / Spark Framework)** | Modern Java frameworks designed for building RESTful, production-grade backend systems quickly. Provide built-in support for security (Spring Security), database access (JPA), and API management.                                                     | Strong fit for the project due to its enterprise-grade reliability, scalability, and strong typing. Ideal for GDPR compliance and secure data handling.                                           | Requires regular dependency updates (Maven/Gradle), JVM upgrades, and server monitoring. Docker and Spring Actuator streamline maintenance. | Higher memory usage compared to lighter frameworks; setup can be complex for MVPs.        |
| **Backend – Python (Flask / Pandas / Matplotlib)** | Flask provides a lightweight backend framework; Pandas handles data analysis; Matplotlib enables data visualization. Ideal for analytics and reporting modules.                                                                                         | Perfect for survey analytics, reporting dashboards, and data visualisation (graphs for progress tracking). Rapid development and flexible.                                                        | Managed through Python virtual environments (venv/Docker). Dependencies updated via pip.                                                    | Limited concurrency and performance for large-scale production loads.                     |
| **Database – MySQL (SQL)**                         | A widely used open-source relational database system supporting transactions, indexing, and JSON fields. Well-known for reliability and wide hosting support.                                                                                           | Good fit for structured data storage such as user accounts, survey data, and rewards.                                                                                                             | Requires schema management, backups, and regular maintenance. Managed services like AWS RDS simplify operations.                            | Slightly weaker JSON and concurrency handling compared to PostgreSQL.                     |
| **Database – PostgreSQL (SQL)**                    | Advanced open-source relational database with full ACID compliance, JSONB support, and efficient query performance.                                                                                                                                     | Excellent for applications that require structured + semi-structured data and analytics capabilities.                                                                                             | Requires periodic backups, schema migrations, and updates via tools like pgAdmin or Docker.                                                 | Risk of corruption if backups aren’t maintained; complex migrations could cause downtime. |
| **Database – Firebase (NoSQL)**                    | Google’s cloud-hosted NoSQL platform offering real-time data sync, authentication, push notifications, and analytics.                                                                                                                                   | Great fit for mobile integrations — handles push notifications, user authentication, and real-time data updates efficiently.                                                                      | Minimal maintenance; fully managed by Google. Regular SDK updates and automatic scaling.                                                    | Vendor lock-in; limited flexibility for complex queries and custom analytics.             |
| **Visualization – Pandas / Matplotlib**            | Python libraries for data manipulation and chart creation (line, bar, scatter, histogram, etc.).                                                                                                                                                        | Essential for visualizing user progress (e.g., symptom scores or survey results).                                                                                                                 | Easy to maintain; version management handled via pip.                                                                                       | Limited interactivity; may need integration with web frameworks for dynamic visuals.      |
